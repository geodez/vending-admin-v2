# 00 Master Brief — Owner OS (общие правила для всех агентов)
Дата: 2026-01-25

Тебе предоставлен архив с ТЗ и документацией проекта Owner OS. Этот архив — единственный источник требований.
Работай строго в рамках своей роли (её задают отдельным rule-файлом).

## Правила
1) Не выдумывай требования. Если чего-то нет в документах — оформи как вопрос/допущение.
2) MVP-first: минимальная реализация, дающая ценность. Не добавляй лишних подсистем.
3) Любые изменения API/моделей/безопасности:
   - зафиксируй в "Decision Log" (что меняем, почему, на кого влияет)
   - отметь затронутые модули (backend/frontend/odoo/devops/qa).
4) Для финансовых действий ассистента действует политика: draft → confirm (по умолчанию).
5) Формат результата (всегда):
   - Deliverables (что сделано/какие файлы/что готово к merge)
   - Checklist (проверки/тесты/ручные сценарии)
   - Risks (риски и как снизить)
   - Questions / Assumptions (вопросы и допущения)
6) Следуй DoD/Repo/Env из документации проекта.

## Стиль
- Коротко и по делу.
- В коде: читаемость, типизация, обработка ошибок, логирование.

## Работа с навыками (Superpowers)
1) **ВСЕГДА** проверяй наличие подходящего навыка (skill) перед выполнением задачи.
   - Планирование/брейншторминг -> `brainstorming`
   - Написание плана -> `writing-plans`
   - Выполнение -> `executing-plans`
   - Отладка -> `systematic-debugging`
   - UI/UX -> `ui-ux-pro-max`
   - TDD (тесты) -> `test-driven-development`
   - Верификация -> `verification-before-completion`
   - Изоляция (worktrees) -> `using-git-worktrees`
   - Завершение ветки -> `finishing-a-development-branch`
2) Используй навык `using-superpowers`, чтобы определить нужный инструмент.
3) Если навык выбран, следуй его инструкциям из файла `SKILL.md` (читай файл через `view_file`).
